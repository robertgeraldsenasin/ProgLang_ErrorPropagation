Repository root: ProgLang_ErrorPropagation

1. Workbook purpose

The workbook is the human-readable experiment notebook for the repository. Machine truth still lives in JSONL logs, but the workbook is where planning, turn review, run roll-ups, and dashboard-level interpretation happen. This package completes all previously placeholder-heavy sheets so that every tab now has a clear function.

The earlier overcount issue came from formula-filled rows being treated like real runs. The repaired workbook now seeds only active planned runs, clears stale working rows when reseeded, and uses simpler dashboard formulas so planned runs and logged runs are counted correctly.

11. Why the analytical variables matter

  • Severity creates an ordered notion of error seriousness so turns can be compared numerically.
  • Improved, regressed, and oscillations are the workbook-side bridge to the paper’s trajectory metrics.
  • first_pass_turn separates quick recoveries from slow or brittle repairs.
  • rows_pred, rows_gold, and symdiff_rows make wrong-result failures interpretable instead of opaque.
  • model_snapshot is critical because browser providers can silently change the active model label.
  • prompt_hash protects the study against undocumented prompt drift.
  • stability rows distinguish robust passes from one-off passes.

The workbook dashboard is a compact tracker. The actual analysis graphs are produced outside the workbook by script 05. The sample smoke charts below show the two headline export types this package generates.

Operationally: script 03 produces JSONL logs; script 04 turns those into workbook rows and formulas; script 05 turns the same logs into paper-facing CSVs and PNG charts. The workbook and the analysis exports therefore stay aligned because they are both downstream of the same machine logs.

13. Proper workbook workflow

  • Seed the workbook before you begin the pilot.
  • Do not change the task pack or planned model roster after the main comparison starts.
  • Sync the workbook after every trajectory or run block.
  • Run stability checks only after Pass.
  • Copy selected evidence to docs/testing-evidence before committing.
  • Use script 09 + script 10 whenever you need to reset the workbook to a clean state.
